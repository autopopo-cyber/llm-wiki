## 一、视觉替代方案

### 1.1 方案对比

| 方案 | 优势 | 劣势 | Orin NX 可行性 | 推荐度 |
|------|------|------|-----------------|---------|
| **YOLOv8-seg + COCO预训练** | 速度快、部署简单 | COCO 缺乏道路场景 | ✅ 完全可行 | ⭐⭐⭐⭐☆ |
| **YOLO-World (开放词汇)** | 无需重新训练，自然语言描述目标 | 速度慢 (15-30ms)、边缘确定性一般 | ⚠️ INT8 可试 | ⭐⭐⭐☆☆ |
| **YOLOP / YOLOPv2** | 三任务一体：检测+可行驶区域+车道线 | 专为自动驾驶设计，机器狗适配需调整 | ✅ FP16 可行 | ⭐⭐⭐⭐☆ |
| **RFNet (RGB-D 异常检测)** | 专注道路异常障碍物 | 需要 RGB-D | ✅ 可行 | ⭐⭐⭐⭐☆ |
| **RELLIS-3D / RUGD** | 户外机器人场景 | 数据集大、训练成本高 | ✅ 可行 | ⭐⭐⭐☆☆ |
| **PatchCore / SimpleNet** | 无需标注，仅需正常场景训练 | 不输出类别，只标记可疑区域 | ✅ 极轻量 | ⭐⭐⭐⭐⭐ |

### 1.2 推荐组合（不依赖朗毅权重）

```
主检测：YOLOv8-seg (COCO预训练) → 人、车、动物等常见类别
辅助：YOLOP / YOLOPv2 → 可行驶区域分割，确定道路范围
党底：LiDAR 地面分割 + 联邦聚类 → 任何物理障碍物
可选：PatchCore / SimpleNet → 视觉可疑区域标记
```

### 1.3 关键开源项目

#### YOLOP / YOLOPv2（三任务）
- 仓库：hustvl/YOLOP / hustvl/YOLOPv2
- 功能：交通目标检测 + 可行驶区域分割 + 车道线检测
- 优势：单模型解决多个感知任务，可行驶区域分割直接告诉机器人哪里能走
- 对四足的价值：可行驶区域 = 避障层的"可行域 polygon"，直接用于边缘约束

#### RFNet (意外障碍物检测)
- 仓库：AHupuJR/RFNet
- 功能：RGB-D 语义分割 + 异常障碍物检测
- 优势：专门针对道路意外障碍（坑洞、碎石、落石等）
- 对四足的价值：户外场景下非标准障碍物检测

#### RELLIS-3D / RUGD (户外语义分割)
- 数据集：RELLIS-3D（澳门大学）、RUGD（Rutgers）
- 仓库：OSSome01/semseg、ankurguria/Semantic-Segmentation-Off-Road-Robotics-RELLIS-3D
- 功能：户外地形、植被、树木、岩石等语义分割
- 对四足的价值：可用于划分可行驶区域与障碍物区域

#### 异常检测（党底视觉）
- PatchCore: hcw-00/PatchCore_anomaly_detection (⭐⭐⭐)
- SimpleNet: jahongir7174/SimpleNet (⭐⭐⭐)
- 功能：仅用正常场景训练，检测离群点/异常模式
- 对四足的价值：轻量级可部署在 Orin NX，作为视觉党底

## 二、LiDAR 党底检测（核心）

### 2.1 原理

不管视觉看到什么，只要 LiDAR 点云里有一个聚类不是地面，就是物理障碍物。这是最简单、最可靠的党底逻辑。

### 2.2 标准流水线

```
LiDAR 点云
    ↓
ROI 剪切（前方感兴趣区域）
    ↓
体素降采样
    ↓
地面分割（RANSAC / 区域增长 / Patchwork++）
    ↓
非地面点云
    ↓
欧氏聚类（带 KD-Tree 加速）
    ↓
聚籿集 → 障碍物列表
```

### 2.3 地面分割算法对比

| 算法 | 原理 | 优势 | 适合场景 |
|------|------|------|---------|
| **RANSAC** | 随机抽样一个平面 | 简单、快速 | 平坦地面 |
| **Region Growing** | 从种子点向外扩展 | 适应小坡度变化 | 小坡度/台阶 |
| **Patchwork++** | 区域化地面估计 | 处理大规模点云、复杂地形 | 户外不平整地面 |
| **LineFit** | 拟合地面线 | 稳健、快速 | 城市道路 |

**Patchwork++** 是当前最佳选择，专为自动驾驶和机器人设计，对坡度、草地、台阶有很好的鲁棒性。

### 2.4 开源工具

| 项目 | 链接 | 功能 |
|------|------|------|
| PCL (Point Cloud Library) | https://github.com/PointCloudLibrary/pcl | 标准点云处理库，包含 RANSAC、聚类、降采样 |
| Patchwork++ | https://github.com/url-kaist/patchwork-plusplus | 地面分割，处理复杂地形 |
| maggieliuzzi/lidar_obstacle_detection | GitHub | 完整流水线：ROI + 降采样 + RANSAC + 聚类 + 检测框 |

## 三、视觉-LiDAR 融合党底

### 3.1 融合策略

```
YOLOv8-seg 检测到的障碍物
    ↓
投影到点云坐标系
    ↓
与 LiDAR 聚籿结果做 ICP/距离匹配
    ↓
合并障碍物列表

如果 LiDAR 有聚籿但 YOLO 没检测到
    → 标记为 "unknown_obstacle"
    → 避障层仍然避让
```

### 3.2 视觉异常区域

如果视觉模型（PatchCore/SimpleNet）检测到可疑区域：
1. 在该区域加强 LiDAR 点云检查
2. 如果 LiDAR 也有点云聚籿 → 确认为障碍物
3. 如果 LiDAR 没有 → 可能是虚景/误报，降低权重

## 四、技术决策

### 最终推荐方案

```
┌────────────────────────────────────────────────┐
│  主检测：YOLOv8-seg (COCO预训练)                    │
│  → 人、车、动物等常见类别检测                          │
├────────────────────────────────────────────────┤
│  辅助：YOLOP / YOLOPv2 可行驶区域分割               │
│  → 确定道路范围，供避障层做边缘约束                    │
├────────────────────────────────────────────────┤
│  党底：Patchwork++ 地面分割 + 欧氏聚类                │
│  → 任何非地面聚籿 = 障碍物，不管是什么类别              │
├────────────────────────────────────────────────┤
│  可选增强：SimpleNet / PatchCore 视觉异常检测        │
│  → 备用，Orin NX 有剩余算力时启用                      │
└────────────────────────────────────────────────┘
```

### 与朗毅 Marathongo 的对比

| 项目 | 朗毅 Marathongo | 本方案 |
|------|-----------------|---------|
| 视觉模型 | YOLO11s-seg (权重未开源) | YOLOv8-seg (COCO预训练，即用) |
| 分割类别 | 人、车、机器人 | 人、车、动物 + 可行驶区域 |
| 视觉-避障融合 | 未完成桥接 | YOLOP 可行域 + LiDAR 聚籿合并 |
| 党底 | 无 | **LiDAR 聚籿必须有** |
