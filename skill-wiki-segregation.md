# 协调者 vs 执行者 — Skill & Wiki 体系区分

## 设计原则

协调者（秦皇）和执行者（始皇/骠骑/丞相）的 skill 和 wiki 内容应有明确区分：

### 协调者（秦皇）— 战略层

**Skills 重点：**
- 任务分发与进度追踪
- A2A 通信协议
- 心跳检测与超时问询
- 组织架构管理
- 理念传播与知识整合

**不应有的 Skills：**
- 具体代码开发流程
- 部署安装步骤
- 硬件驱动安装

**Wiki 重点：**
- 组织架构与节点状态
- 理念文档（ORIGIN.md, 仙秦帝国）
- 任务进度表
- Agent 协作协议
- 交接记录

### 执行者 — 执行层

**Skills 重点：**
- 开发与部署具体流程
- 工具安装与配置
- 代码审查与测试
- 硬件驱动与 CUDA 管理

**Wiki 重点：**
- 技术笔记与踩坑记录
- 代码片段与解决方案
- 环境配置详情
- 依赖关系图

## 同步机制

| 内容类型 | 协调者 | 执行者 | 同步方式 |
|----------|--------|--------|----------|
| 理念文档 | ✅ 维护 | ✅ 只读 | git sync |
| 技术笔记 | ✅ 只读 | ✅ 维护 | git sync |
| 任务进度 | ✅ 维护 | ✅ 上报 | API 通信 |
| 安全策略 | ✅ 维护 | ✅ 执行 | git sync |
| Soul.md | ✅ 审批 | ✅ 自述 | API 通信 |

## Git 仓库规划

```
~/llm-wiki/          # 共享 wiki（git sync）
├── xianqin-*.md     # 仙秦帝国相关（协调者维护）
├── tech-*.md        # 技术笔记（执行者维护）
├── daily-digest.md  # 每日摘要
└── plan-*.md        # Plan-tree offload

~/.hermes/skills/    # 各自独立
├── coordinator/     # 协调者专属 skills
└── executor/        # 执行者专属 skills
```
