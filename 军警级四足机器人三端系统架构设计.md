## 一、通信架构：双模式解耦设计

核心原则：**通信模块与业务逻辑完全解耦**，同一套业务代码可在无线/光纤两种模式下无缝运行。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           通信抽象层 (Communication Abstraction)              │
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────────┐ │
│  │                      统一消息接口 (protobuf)                              │ │
│  │  RobotState / Command / Task / Telemetry / VideoFrame                     │ │
│  └─────────────────────────────────────────────────────────────────────────┘ │
│                                    │                                        │
│         ┌──────────────────────────┼──────────────────────────┐             │
│         │                          │                          │             │
│    ┌────┴────┐              ┌────┴────┐              ┌────┴────┐          │
│    │ 方式A   │              │ 方式B   │              │ 方式C   │          │
│    │无线模式 │              │光纤模式 │              │混合模式 │          │
│    └────┬────┘              └────┬────┘              └────┬────┘          │
│         │                          │                          │             │
└─────────┼──────────────────────────┼──────────────────────────┼─────────────┘
          │                          │                          │
    点对点射频/5G                 光纤+中继                   自动切换
```

### 方式A：无线模式（常规/民用）

| 链路 | 技术 | 用途 | 距离 | 特点 |
|------|------|------|------|------|
| 图传链路 | 900MHz / 2.4G / 5.8G 点对点 | 实时视频 + 遥控指令 | 1-5km | 低延迟、抗干扰一般 |
| 数据链路 | 5G 公网/专网 | 状态上报、任务下发、地图数据 | 无限 | 依赖基站覆盖 |
| 备份链路 | 4G LTE | 5G 断网时降级 | 无限 | 带宽窄但覆盖广 |

**适用场景：** 城市巡逻、园区安防、公开活动安保。

### 方式B：光纤模式（军警/矿井/电磁对抗）

| 链路 | 技术 | 用途 | 距离 | 特点 |
|------|------|------|------|------|
| 主链路 | 单模光纤 + 光电转换模块 | 全双工数据 + 视频 | 10km+ | 电磁免疫、带宽大、保密 |
| 中继节点 | 光纤交换机 / 便携式中继箱 | 光纤延长 + 分叉接入 | 每 10km 一跳 | 野外部署、即插即用 |
| 卫星备份 | 北斗短报文 / 天通卫星 | 无光纤时的极简状态回传 | 全球 | 速率极低，仅关键状态 |

**光纤接入点设计：**

```
机器狗尾部 ── 光纤绞盘（自动收放） ── 便携式光纤终端盒
                                      │
                              ┌───────┴───────┐
                              ▼               ▼
                         中继箱（野外）      指挥车/控制室
                         （光电转换+交换）   （大屏调度系统）
                              │
                              └──────→ 卫星上行（极端情况）
```

**适用场景：** 矿井救援（井下无电磁信号）、电磁对抗环境（ Jamming）、重要设施内部（防窃听）。

### 通信抽象层接口设计

```protobuf
// 统一消息格式，与传输方式无关
message Envelope {
  string msg_id = 1;        // UUID，用于去重和ACK
  double timestamp = 2;     // 发送端时间戳
  string robot_id = 3;
  MsgType type = 4;         // STATE / COMMAND / TASK / VIDEO / LOG
  bytes payload = 5;        // 加密后的 protobuf 消息
  bytes signature = 6;      // HMAC-SHA256 签名
  int32 ttl = 7;            // 生存时间（光纤模式可设长）
}

// 通信适配器接口（C++ / Python）
class CommAdapter {
  virtual bool Send(const Envelope& env) = 0;
  virtual bool Receive(Envelope* env, int timeout_ms) = 0;
  virtual LinkStatus GetLinkStatus() = 0;  // 带宽/延迟/丢包率
  virtual bool IsEncrypted() = 0;
};

// 三种实现
class RadioAdapter : public CommAdapter { /* 900M/2.4G/5.8G + 5G */ };
class FiberAdapter : public CommAdapter { /* 光纤 + 光电转换 */ };
class HybridAdapter : public CommAdapter { /* 自动切换最优链路 */ };
```

### 军警级安全设计

| 层级 | 措施 | 实现 |
|------|------|------|
| **链路加密** | AES-256-GCM | 每会话动态密钥，前向安全 |
| **身份认证** | 双向证书 + HMAC | 机器人/APP/云端三方互认 |
| **消息防重放** | 时间窗口 + nonce | 5分钟窗口，过期丢弃 |
| **物理安全** | 光纤防折断检测 | 光功率监测，断纤自动报警+录像本地保存 |
| **抗干扰** | 跳频（射频）/ 电磁免疫（光纤） | 射频：900MHz 窄带跳频；光纤：天然免疫 |

## 三、APP 端设计（手持/车载终端）

### 功能模块

| 模块 | 功能 | 技术 |
|------|------|------|
| **地图显示** | 机器人实时位置、历史轨迹、waypoints、任务路线 | 高德 SDK（在线）/ MBTiles（离线） |
| **机器人列表** | 在线状态、电量、信号强度、当前任务、传感器健康 | MQTT/光纤协议 + 心跳检测 |
| **实时画面** | 第一视角视频、点云可视化（2D/3D）、热成像（可选） | H.265 硬解、PCL 点云渲染 |
| **任务控制** | 点选导航、巡逻路线、跟随模式、编队模式、急停召回 | Playbook 下发 + 即时指令 |
| **通信管理** | 链路切换（5G/光纤/卫星）、带宽监测、加密状态 | 通信适配器状态回调 |

### 遥控手柄模式

APP 支持两种交互方式：

1. **触屏模式**：大屏显示，点选操作，适合静态指挥
2. **手柄模式**：左摇杆平移、右摇杆旋转、按键急停/召回，适合伴随行进

手柄直接通过蓝牙/WiFi 连接 APP，不直接连接机器狗（统一走通信适配器）。

## 四、机器人端设计（Unitree A2）

### 硬件利用

A2 自带 **Intel Core i7 + 独立显卡**，算力远超 Go2 的 ARM 板。可以直接在机载运行：

| 模块 | 算力需求 | A2 i7 可行性 |
|------|---------|-------------|
| glio_mapping (SLAM) | CPU 密集型 | ✅ 轻松运行 |
| vo_navigation (避障) | CPU + 内存 | ✅ 轻松运行 |
| YOLO11s-seg (TensorRT) | GPU | ✅ 独立显卡可跑 FP16 |
| Qwen2.5-VL (VLM) | GPU | ✅ i7 + 独显可跑 7B INT4 |
| 视频编码上传 | GPU NVENC | ✅ 硬编码 H.265 |

**不需要额外买 Jetson！** A2 的 i7 足够跑完整导航+视觉栈。

### 与宇树 SDK 的对接

基于真实 SDK 代码（`unitree_sdk2/example/a2/sport/`）：

```cpp
// 1. 初始化 SDK
unitree::robot::ChannelFactory::Instance()->Init(0, "eth0");
unitree::robot::a2::SportClient sport_client;
sport_client.SetTimeout(25.0f);
sport_client.Init();

// 2. 导航层输出 → 统一控制接口
// vo_navigation 输出 cmd_vel = (vx, vy, vyaw)
sport_client.Move(vx, vy, vyaw);  // 直线：vx=0.5, vy=0, vyaw=0

// 3. 急停/召回
sport_client.StopMove();           // 停止
sport_client.BalanceStand();       // 平衡站立
sport_client.Damp();               // 阻尼（倒下保护）

// 4. 状态获取（ROS2 topic 或 SDK 订阅）
// /sportmodestate: position[3], velocity[3], imu_state, foot_force[4]
// /lowstate: 500Hz motor/imu/battery
// /utlidar/cloud: 点云数据
```

## 五、云端/指挥所设计

### 部署形式

| 场景 | 部署方式 | 规模 |
|------|---------|------|
| 车载指挥 | 加固服务器（2080Ti）+ 便携大屏 | 1-4 台机器狗 |
| 固定指挥所 | 机架服务器 + 大屏墙 | 10+ 台机器狗 |
| 野外临时 | 便携箱式服务器 + 折叠屏 | 2-4 台机器狗 |

### 核心服务

| 服务 | 技术 | 功能 |
|------|------|------|
| Playbook 引擎 | Python + 状态机 | 任务编排、步骤依赖、超时/失败重试 |
| 地图服务 | PostGIS + 高德 API | 路径规划、坐标转换、离线地图瓦片 |
| 视频汇聚 | SRS / MediaMTX | 多路视频流转发、录像、时移回放 |
| 时序数据库 | InfluxDB / TDengine | 传感器数据持久化、趋势分析 |
| 告警中心 | Prometheus Alertmanager | 分级告警、通知推送、联动响应 |

## 六、与宇树原厂能力的衔接

A2/Go2 原厂已提供的能力，我们直接复用：

| 原厂能力 | SDK/ROS2 接口 | 我们的用法 |
|---------|--------------|-----------|
| RL 步态控制 | `SportClient::Move()` | 导航输出 cmd_vel → Move() |
| 内置 IMU | `/lowstate` 500Hz | SLAM 去畸变、状态监控 |
| 内置 LiDAR | `/utlidar/cloud` | glio_mapping 输入 |
| 遥控器 | `/wirelesscontroller` | 手动接管、急停 |
| 足端力传感器 | `/lowstate` foot_force | 地面检测、滑移判断 |
| 电池管理 | `/lowstate` power_v/a | 电量监控、低电量召回 |

**我们不重新发明：** 步态控制、底层电机驱动、IMU 驱动、电池管理。这些宇树已做好。

**我们重点做：** 导航避障、视觉感知、通信安全、任务调度、多机协同。

## 七、技术决策记录

| 日期 | 决策 | 理由 |
|------|------|------|
| 2026-04-21 | 通信模块解耦（Radio/Fiber/Hybrid） | 军警场景要求电磁免疫+矿井可用，光纤是刚需 |
| 2026-04-21 | A2 i7 直接跑导航+视觉，不买 Jetson | i7+独显算力足够，减少硬件复杂度 |
| 2026-04-21 | 宇树 SDK `Move()` 作为统一控制接口 | 不碰 RL 运控黑盒，导航只管 cmd_vel |
| 2026-04-21 | 云端可选，APP-机器人可点对点 | 光纤模式下指挥车就是"云端"，不需要公网 |
| 2026-04-21 | ROS2 Humble + CycloneDDS | 宇树官方支持，A2 兼容 Go2 接口 |
