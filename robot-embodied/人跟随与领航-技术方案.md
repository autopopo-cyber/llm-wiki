## 概述

人跟随（Human Following）是四足机器人的核心应用场景之一。与导航-避障系统不同，跟随任务不需要预先知道目的地，而是**以人类为动态目标**进行实时跟踪。系统应在本地闭环完成全部计算，不强制依赖网络或后台规划。

本文档涵盖两种工作模式：
1. **跟随模式**：机器人走在人类后方/侧方
2. **领航模式**：机器人走在人类前方（需预知目的地或预先探索）

## 二、目标检测与识别

### 2.1 视觉检测

| 方案 | 模型 | 速度 | 说明 |
|------|------|------|------|
| 行人检测 | **YOLOv8-pose / YOLO11-pose** | 30-60FPS | 检测人体 + 关键点，可识别朝向 |
| 人员再识别 | **OSNet / FastReID** | 实时 | 锁定特定人员（衣着/体型特征） |
| 深度估计 | **Intel RealSense D435** | 30FPS | 直接输出深度，无需单目深度估计 |

**ReID（行人再识别）策略：**
- 首次见面时提取特征向量（衣着颜色、背包、体型）
- 后续帧用余弦相似度匹配，即使短暂遮挡也能找回
- 特征库只保留最近 N 帧，防止漂移

### 2.2 LiDAR 行人检测

LiDAR 对行人的检测不依赖光照，是视觉的有力补充。

**人腿模式识别（Leg Pattern Detection）：**
```
LiDAR 水平扫描 → 地面分割（Patchwork++）→ 
剩余点聚类 → 筛选双柱体模式（间距 10-40cm，直径 5-15cm）→
行人候选 → 时序跟踪（最近邻/卡尔曼）
```

**特点：**
- 在人群中可区分多个独立行人（每对人腿一个 ID）
- 夜间/强光/背光场景下稳定工作
- 可估计行人朝向（通过双腿前后关系）

### 2.3 视觉-LiDAR 融合跟踪

**融合策略：紧耦合跟踪器**

| 传感器 | 提供信息 | 优势场景 |
|--------|---------|---------|
| 视觉 | 外观特征、ReID、人体朝向 | 人群区分、重识别 |
| LiDAR | 精确距离、速度、人腿位置 | 测距准确、夜间可用 |

**实现：**
- 视觉检测框投影到 LiDAR 点云 → 获取精确深度
- LiDAR 聚类反投影到图像 → 验证视觉检测
- 卡尔曼滤波状态： `[x, y, vx, vy]`，观测融合视觉位置 + LiDAR 速度

**开源方案：**
- **Lidar-Camera Fusion (LC-Fusion)**：https://github.com/robotic-vision
- **Multiple Object Tracking (mmTracking)**：mmdetection + mmtracking

## 三、跟随控制器

### 3.1 跟随模式（Follow Mode）

机器人走在人类后方/侧方，保持固定相对位姿。

**控制律（极坐标系）：**

```
目标：保持 (d, α) = (d_target, 0)

d：机器人到目标的距离
α：目标在机器人坐标系中的方位角
θ：目标朝向与机器人朝向的夹角

v = k_v · (d - d_target) + k_vd · d_dot
ω = k_α · α + k_θ · θ + k_ω · ω_target

其中 ω_target 从目标速度预测（补偿延迟）
```

**安全距离策略：**

| 场景 | 距离 d_target | 说明 |
|------|--------------|------|
| 室内/拥挤 | 1.0m | 近距跟随，避免挡路 |
| 室外平地 | 1.5-2.0m | 标准跟随距离 |
| 人群密集 | 2.5m+ | 留足反应空间 |

**丢失处理：**
1. **短时丢失（<2s）**：保持最后已知位置，预测性搜索
2. **长时丢失（>2s）**：原地旋转扫描，或回到最后已知位置等待
3. **完全丢失**：停止并发出提示音/灯光

### 3.2 领航模式（Lead Mode）

机器人走在人类前方，起引导作用。

**两种实现：**

**A. 已知目的地（导航+引导）**
```
人类说"带我去图书馆" → VLM 理解 → 
查询本地地图/高德 API 获取路径 → 
机器人走在前方 3-5m → 到达后停下等待
```

**B. 未知目的地（探索+跟随反转）**
```
机器人先探索前方（自主导航）→ 
识别安全路径 → 走在前方 → 
人类跟上 → 遇岔路停下等待指令
```

**领航时的避障优先级：**
1. 保障人类路径安全（清除/标记前方障碍）
2. 保持可视线联系（不拐急弯，不被遮挡）
3. 在岔路口/电梯口停下等待确认

## 四、与导航避障系统的集成

人跟随不是替代导航避障，而是**动态目标层叠加在现有系统上**。

```
┌─────────────────────────────────┐
│        目标优先级仲裁层           │
├─────────────────────────────────┤
│  模式 1：自由导航                │
│    目标 = waypoints              │
│                                  │
│  模式 2：跟随                    │
│    目标 = 跟踪器输出的人类位置     │
│    避障层仍然运行（绕开其他障碍）   │
│                                  │
│  模式 3：领航                    │
│    目标 = 前方路径点              │
│    速度降低，等待人类跟上           │
└─────────────────────────────────┘
```

**关键：** 避障层（vo_navigation）始终运行，不管目标是 waypoints 还是人类。跟随任务只是改变"目标点"的来源。

## 六、MuJoCo 验证方案

| 测试场景 | 设置 | 通过标准 |
|----------|------|---------|
| 直线跟随 | 人类走直线，机器人在后 1.5m | 距离误差 < 0.3m，无碰撞 |
| 转弯跟随 | 人类急转弯（90°） | 机器人不丢失目标，延迟 < 1s |
| 人群干扰 | 3-5 个干扰行人交叉 | ReID 正确锁定，不误跟 |
| 遮挡恢复 | 障碍物遮挡目标 2s | 2s 内重新锁定原目标 |
| 避障+跟随 | 前方出现静态障碍 | 绕行后回到跟随位置 |
| 夜间模拟 | 关闭视觉，仅用 LiDAR | 纯 LiDAR 跟随稳定 |

## 七、技术决策记录

| 日期 | 决策 | 理由 |
|------|------|------|
| 2026-04-20 | 人跟随不强制依赖网络 | 使用场景（校园/公园）网络不稳定 |
| 2026-04-20 | 不强制佩戴设备 | 降低用户门槛，纯被动感知 |
| 2026-04-20 | 视觉+LiDAR 融合跟踪 | 单视觉在夜间/背光失效；单 LiDAR 无外观特征 |
| 2026-04-20 | 领航模式需要本地地图 | 无网络时依赖预缓存地图或离线导航 |
