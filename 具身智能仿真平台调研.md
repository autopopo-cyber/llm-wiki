## 选型总体策略

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| **纯自动驾驶** | CARLA / SUMO | 道路网络最完整，交通流最丰富 |
| **纯机器人（室内+简单室外）** | Gazebo Ignition / Webots | 传感器最全，ROS对接最成熟 |
| **端到端RL训练** | MuJoCo / Brax | 速度最快，无GPU也能百万step |
| **多机协作、空地协同** | Webots / Gazebo Ignition | 支持异构体，协议最通用 |
| **轻量化+地形+道路** | **Gazebo Ignition** | 四项都覆盖，最均衡 |

## 1. 自动驾驶专用

### 1.1 CARLA（推荐：纯三维自动驾驶）

| 维度 | 详情 |
|------|------|
| 物理引擎 | PhysX (Unreal Engine 4) |
| 地形/道路 | 内置 Town 系列地图（最大 12km²），OpenDRIVE 道路网络，支持自定义 |
| 多智能体 | ✅ 支持 |
| 传感器 | LiDAR/Radar/GPS/IMU/Camera/Semantic Segmentation（UE内置） |
| 学习曲线 | 陡峭（UE4 编译 30GB+，Python API 需加载世界） |
| 资源占用 | 32GB 内存，RTX 3070+ |
| 许可证 | MIT（开源），UE4 需 EULA |
| 与 Isaac 对比 | 比 Isaac 重很多（UE4 是个大怪兽），但道路网络 CARLA 最强 |

**适合场景**：有UE4经验，需要高质量视觉的AD算法训练

### 1.2 SUMO（推荐：交通流+协同决策）

| 维度 | 详情 |
|------|------|
| 物理引擎 | 无三维物理，宏观交通流模型（Krauss/IDM） |
| 地形/道路 | OpenStreetMap 直接导入，支持大规模路网（城市级） |
| 多智能体 | ✅ 万级车辆并发 |
| 传感器 | 无感官模拟，只有交通状态（速度、位置、车道） |
| 学习曲线 | 平缓（纯Python，轻量） |
| 资源占用 | 极低（CPU即可） |
| 许可证 | EPL-2.0（开源） |

**与 CARLA 协同**：SUMO 管决策/交通，CARLA 管视觉/物理。

### 1.3 MetaDrive（推荐：RL + Procedural Generation）

| 维度 | 详情 |
|------|------|
| 物理引擎 | Custom |
| 地形/道路 | 程序化生成无限赛道（弯道、交叉口、障碍物） |
| 多智能体 | ✅ 支持 |
| 传感器 | RGB/Depth/Lidar/BEV |
| 学习曲线 | 平缓（pip install metadrive，纯Python） |
| 资源占用 | 轻量（`Rendering=False` 纯Panda3D） |
| 许可证 | Apache-2.0 |
| 特色 | **无限程序化生成**，RL训练神器 |

### 1.4 SMARTS

| 维度 | 详情 |
|------|------|
| 物理引擎 | Custom |
| 地形/道路 | SUMO路网 + SUMO交通模型 |
| 多智能体 | ✅ Multi-agent RL 专用 |
| 许可证 | MIT/BSD |
| 特色 | 多智能体交互模拟（敌意车辆、社会车辆） |

## 2. 机器人通用（含室外）

### 2.1 Gazebo Ignition（推荐：最均衡）

| 维度 | 详情 |
|------|------|
| 物理引擎 | **可选**：ODE / Bullet / DART / TPE（自定义地形） |
| 地形/道路 | **程序化地形**（Perlin Noise 高度图）；可导入 DEM 数字高程模型；插件支持里程计、纹理 |
| 多智能体 | ✅ 多实体、多机器人独立物理 |
| 传感器 | LiDAR/Camera/IMU/Depth/Odometry/Force sensor |
| 学习曲线 | 中等（SDF 格式，ROS2 原生集成） |
| 资源占用 | 中等（比 Isaac 轻，支持 Headless） |
| 许可证 | Apache-2.0 |

**关键优势**：
- **地形模拟最强**：可加载真实 DEM 数据模拟山地、丘陵
- **LOD 地形插件**：基于高度的程序化地形生成，适用于大规模四足训练
- **ColeTerrainPlugin**：GPU-accelerated 地形碰撞检测

### 2.2 Webots（推荐：新手最快上手）

| 维度 | 详情 |
|------|------|
| 物理引擎 | ODE |
| 地形/道路 | 地形节点（高度图）、Road 节点（含车道线、人行道） |
| 多智能体 | ✅ TCP/socket 控制器 |
| 传感器 | 全部，含坐标系转换自动处理 |
| 学习曲线 | 最平缓（开箱即用，自带 500+ 场景） |
| 资源占用 | 轻量 |
| 许可证 | Apache-2.0 |

**最大优势**：One-click 开箱，适合快速验证。不适合大规模 RL（比 MuJoCo 慢）。

### 2.3 MuJoCo（推荐：端到端RL训练最快）

| 维度 | 详情 |
|------|------|
| 物理引擎 | **自研**（凸几何碰撞+软接触） |
| 地形/道路 | 程序化：Plane(无限平地) / HField(高度图) / 凸mesh |
| 多智能体 | ✅ 独立实体 |
| 传感器 | 需手动模型（Camera/Lidar = Ray casting 显式） |
| 学习曲线 | 中等（MJCF/URDF，Python 绑定） |
| 资源占用 | **极低**（CPU 百万 step/s） |
| 许可证 | Apache-2.0（Google DeepMind 开源） |

**最大优势**：步进速度最快。2023年支持 Native Rendering（OpenGL），无需单独显卡跑感官。可与 Brax(JAX) 结合，TPU 上每秒千万级 step。

### 2.4 PyBullet（推荐：兼容最多URDF）

| 维度 | 详情 |
|------|------|
| 物理引擎 | Bullet |
| 地形/道路 | `loadURDF("plane.urdf")` + `createMultiBody` 自定义 |
| 多智能体 | ✅ 独立实例 |
| 传感器 | `getCameraImage`，Ray casting 手动 |
| 资源占用 | 极低 |
| 许可证 | zlib |
| 状态 | ⚠️ 维护放缓，一大半模型朝 MuJoCo 迁移 |

### 2.5 O3DE (Open 3D Engine) + ROS2

| 维度 | 详情 |
|------|------|
| 物理引擎 | PhysX 5 |
| 特色 | 开源版 Lumberyard（AWS），高质量渲染 |
| 学习曲线 | 陡峭 |
| 适合 | 对渲染质量有要求，不想用 Unity/UE4 的团队 |

### 2.6 Unreal + AirSim

⚠️ **项目已停止维护**（Microsoft 2022年归档）。IAMAI/DARPA 接手了一个分支叫 Project AirSim，但已不是原版。

原版 AirSim 特点：Unreal 插件式，in-the-loop with PX4/ArduPilot，适合无人机+车。

## 3. 机器学习专用加速框架

### 3.1 Isaac Sim (NVIDIA)

| 维度 | 详情 |
|------|------|
| 物理引擎 | PhysX 5 |
| 特色 | GPU-accelerated 物理（百万刚体并发）、SDG（合成数据生成）、cuOpt（最优路径） |
| 资源占用 | 重型（32GB+，RTX 3080+） |
| 许可证 | NVIDIA License |

**与 Issac Gym 区别**：Issac Gym 是纯RL训练框架（PyTorch），无仿真只是物理引擎。

### 3.2 Isaac Lab（推荐：当确定用 NVIDIA 生态时）

| 维度 | 详情 |
|------|------|
| 特色 | Isaac Sim + Gym 联合训练框架，原生支持 RL/Imitation Learning |
| 优势 | GPU parallel env 训练 |

## 4. 地形模拟专题对比

| 方案 | 真实 DEM 支持 | 程序化地形 | 多智能体物理 | 速度 |
|------|-------------|-----------|-------------|------|
| **Gazebo Ignition** | ✅ (DEM SDF) | ✅ (Perlin plugin) | ✅ (ODE/Bullet/DART) | 中 |
| MuJoCo | ❌ (HField 可等效) | ✅ (HField) | ✅ (自定义凸mesh) | **最快** |
| Webots | ✅ (ElevationGrid) | ✅ (原语拼接) | ✅ (ODE) | 中 |
| Isaac Sim | ✅ 早期支持 | ❌ 弱 | ✅ (PhysX 5) | 慢 |
| CARLA | ❌ (固定Town) | ❌ | ✅ (PhysX) | 慢 |

## 5. 最终推荐（针对你的四足导航项目）

### 阶段一：快速原型验证

```
Webots 或 Gazebo Ignition
```
- 有现成四足机器人模型（Uniro, Ant, ANYmal）
- 快速跑通传感器 pipeline

### 阶段二：场景扩展+地形

```
Gazebo Ignition (+ DEM 地形)
```
- 加载真实山区 DEM 数据
- ColeTerrainPlugin GPU 加速碰撞
- ROS2 原生，导航对接无缝

### 阶段三：端到端RL训练

```
MuJoCo (body 级) + Warp/Diffrax（ physics可微）
  或
Gymnasium + Isaac Lab (NVIDIA生态)
```

### 阶段四：多机协作、空地协同

```
Gazebo Ignition + 自定义 SDF（异构体）
```

## 关键快速决策

| 问法 | 答案 |
|------|------|
| 要渲染质量（VLM训练）？ | Isaac Sim / CARLA |
| 不要求渲染，只要物理正确？ | MuJoCo |
| 要真实地形（山地/野外）？ | Gazebo Ignition + DEM |
| 要道路网络（城市导航）？ | CARLA + OpenDRIVE / SUMO |
| 要开盒即用？ | Webots |
| 要最快RL收敛？ | MuJoCo / Brax |
| 要 ROS2 原生？ | Gazebo Ignition |
| 要 GPU 并行百万 step？ | Isaac Lab / Isaac Gym |
