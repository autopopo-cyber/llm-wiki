## 二、激光雷达如何克服足式机器人晃动

### 2.1 核心机制：ESKF + IMU 前向传播 + 点云反向去畸变

Marathongo 没有针对足式机器人的专门步态补偿，而是采用通用的 **LiDAR-Inertial 紧耦合 SLAM** 方案：

```
IMU (200Hz+) → ESEKF 前向传播 → 每时刻位姿保存
                                    ↓
LiDAR 点云 → 逐点查找对应 IMU 位姿 → 反向补偿到帧末坐标系
```

**关键代码路径：**

| 步骤 | 文件 | 行号 | 机制 |
|------|------|------|------|
| IMU 初始化 | `imu_processing.cpp` | 118-257 | 统计前10帧均值，RTK辅助初始航向 |
| ESEKF 前向传播 | `imu_processing.cpp` | 297-371 | 每IMU时刻predict，保存位姿到IMUpose向量 |
| 多传感器融合更新 | `glio_estimator.cpp` | 119-196 | GNSS/轮速/角度传感器按时机更新 |
| 点云去畸变 | `imu_processing.cpp` | 384-420 | 反向遍历点云，插值IMU位姿，补偿到帧末 |
| ZUPT 零速检测 | `glio_estimator.cpp` | 249-271 | 加速度/角速度极差判断零速 |

**去畸变核心公式：**
```cpp
R_i = R_imu * Exp(angvel_avr, dt);          // 当前时刻旋转
T_ei = pos_imu + vel_imu*dt + 0.5*acc*dt² - imu_state.pos;  // 平移偏移
P_comp = R_L_I^T * (R_imu_e^T * (R_i * (R_L_I*P_i + T_L_I) + T_ei) - T_L_I)
```

### 2.2 四足 vs 双足：差异比预期小

**核心判断：四足晃动幅度小于双足，但频率约为双足2倍。** Marathongo 原始参数（针对双足）对四足基本适用，无需大幅调整。

| 参数 | Marathongo 原值 | 四足建议 | 理由 |
|------|----------------|---------|------|
| acc_cov | 0.01 | **保留 0.01** | 四足冲击比双足小，不需要放大 |
| LASER_POINT_COV | 0.001 | **保留 0.001** | 晃动幅度小，点云误差可控 |
| ZUPT | 启用 | **需评估** | 四足步态无真正零速时段，但双足也没有——需实测验证是否误触发 |
| IMU 频率 | 200Hz | **保留 200Hz** | 四足晃动频率更高(2x)但幅度更小，200Hz 足够捕获 |
| IMU bias | ESEKF 标准方案 | **保留** | 四足振动不会产生比双足更大的 bias 漂移 |

**⚠️ 唯一需关注点：** 四足步态频率约为双足2倍，意味着 IMU 信号的高频分量更丰富。ESEKF 的过程噪声 Q 矩阵可能需要微调，但量级不变。建议先跑原始参数，再根据实测数据调整。

## 三、高速运动避障与局部寻路

### 3.1 双避障系统架构

项目包含**两套并行避障系统**，针对不同场景：

| 系统 | 模块 | 场景 | 复杂度 |
|------|------|------|--------|
| **系统A** | `marathontracking/local_planner` | 复杂赛道（有预录路径） | 高 |
| **系统B** | `tangent_arc_navigation/vo_navigation` | 极简路线（切线圆弧） | 中 |

### 3.2 系统A：局部规划器

**三模式状态机：**
```
TRACKING (快速冲刺) → AVOIDANCE (快速旋转) → RECOVERY (低速恢复)
```

| 模式 | 线速度范围 | 角速度范围 | 碰撞半径 | 触发条件 |
|------|-----------|-----------|---------|---------|
| TRACKING | 2.5-3.5 m/s | ±2.3 rad/s | 0.6m | 前方无障碍 |
| AVOIDANCE | 1.0-2.3 m/s | ±1.5 rad/s | 0.6m | 路径受阻 |
| RECOVERY | 0-0.6 m/s | ±0.8 rad/s | 1.0m | 卡死恢复 |

**核心流程：**
1. **体素地图构建** — `RingVoxelMap<GridElevation>` 20m×20m×3m / `RingVoxelMap<GridObstacle>` 20m×20m（分辨率0.2m）
2. **路径采样** — 加载A000-A019共20条预录路径，按当前位姿变换到世界坐标
3. **碰撞检测** — 正方形碰撞体扫掠，碰撞半径按模式区分
4. **Dijkstra/A\* 搜索** — 26连通体素，RECOVERY模式使用
5. **速度输出** — 运动学包络裁剪 + 模糊/RL控制

### 3.3 系统B：切线圆弧 VO 避障（2457行核心代码）

**核心思想：Velocity Obstacle + 圆弧轨迹碰撞模拟**

```
1. 前方梯形区域障碍检测 → 速度相关梯形（速度越快越宽）
2. 圆弧轨迹碰撞模拟 → 对每个障碍物模拟不同角速度下的圆弧
3. 切线方向选择 → 找到碰撞圆弧的切线方向作为避障航向
4. 最优航向选择 → 左右两侧分别搜索，选最接近目标方向的
5. 速度规划 → 基于最近障碍距离 + LPF平滑 + 加速度限制
```

**速度规划关键参数：**
- 加速度限制：上升 1.0 m/s²，下降 2.5 m/s²
- 角速度限制：CSV查找表 `vo_angular_limits.csv`
- 状态机：NORMAL / ODOM_UNHEALTHY / RECOVER_OUTSIDE_LANES

### 3.4 四足适配要点

| 问题 | Marathongo方案 | 四足需要 |
|------|---------------|---------|
| 运动学模型 | 差分驱动假设 | 全向移动模型（四足可侧移） |
| 碰撞体 | 正方形扫接 | 需改为四足步态包络 |
| 预录路径 | A000-A019 | 需重新生成（曲率/速度不同） |
| 速度范围 | 最高3.5m/s | 四足通常0.5-2.0m/s |
| ZUPT | 零速判断 | 四足需禁用 |
| **可行域约束** | **无** | **必须新增** |

**关键发现：Marathongo 完全没有"可行域"概念。**

`vo_navigation` 的避障逻辑是：遇到障碍物 → 检测可行方向 → 切弧绕行。这个逻辑在**开放场地**（无边界限制）是成立的，但在**赛道/道路场景**中会冲出边界。

**2026 也庄半马事故：机器人往路边障碍物外侧绕，绕出了赛道。**

**四足导航必须新增的约束：**

```
导航层→避障层：道路可行域 polygon（左右边界）

避障层内部逻辑：
  检测障碍物 → 计算绕行方向 → 判断是否在可行域内 → 仅向可行方向切弧
                                          │
                                          └→ 两侧都被挡 → 减速停止/报警
```

**具体实现：**

| 约束项 | 方法 | 位置 |
|--------|------|------|
| 可行域表示 | 道路左右边界的 polyline | 导航层生成，传给避障层 |
| 切弧有效性检查 | 切弧路径全程与可行域相交 | 避障层内部 |
| 无可行方向时 | 减速停止、视觉提示、电子圈提示 | 避障层决策 |

**MUJoCo 验证用例：**

| 测试场景 | 障碍物 | 期望行为 | 关键检查 |
|----------|-------|---------|----------|
| 左壁撞 | 左边界+障碍物 | 向右绕，不冲出右边界 | 绕行方向在可行域内 |
| 右壁撞 | 右边界+障碍物 | 向左绕，不冲出左边界 | 绕行方向在可行域内 |
| 两侧夹击 | 两侧都有障碍物 | 减速停止/报警 | 不强行穿越 |
| 盲角突发 | 左视觉盲区 | 检测到后停止 | 不冲出边界 |

## 四、导航地图路线规划

### 4.1 现状：仅支持预录路径，不支持 OSM/高德

**❌ 不支持 OpenStreetMap / 高德地图 / 百度地图**

项目没有全局规划器，路线来源是**预录路径**：

| 路径类型 | 格式 | 用途 |
|---------|------|------|
| `.path` 文件 | 逗号分隔 x,y,z (FLU坐标系) | local_planner 路径采样（A000-A019） |
| `free_path.txt` | 空格分隔 timestamp x y z qx qy qz qw | 仿真/实车自由路径 |
| `/original_path` | ROS topic | path_process_node 跟踪目标 |

**path_process_node 功能：**
- 接收预录全局路径
- 计算跟踪误差：`target_x`(4.9m前视), `error_angle`, `curvature_front`, `curvature_back`
- 前视距离：`lookahead_time=5s × velocity`，范围 [20m, 20.1m]
- 输出给模糊/RL控制器

**路径偏移生成：**
- 中心线 → 左右偏移 ±3.5m → 赛道边界

### 4.2 四足导航需要补充的全局规划

| 需要的模块 | 说明 | 可选方案 |
|-----------|------|---------|
| **OSM 地图解析** | 解析 OpenStreetMap 路网 | `osmnx` (Python) / `libosmium` (C++) |
| **高德地图 API** | 实时路况 + 人行道数据 | 高德开放平台路径规划API |
| **全局路径规划** | A*/Dijkstra on 拓扑路网 | Nav2 planner_server + OSM插件 |
| **拓扑→度量映射** | 路网节点→局部规划目标 | 需自研（Marathongo无此模块） |
| **室内外切换** | GNSS退化检测 + 地图衔接 | 需自研 |

## 五、未开源/缺失部分清单

| 模块 | 状态 | 重要性 | 说明 |
|------|------|--------|------|
| **MPC 控制器** | ⚠️ 代码完整但 disable | 低 | OSQP QP求解，差分驱动模型，N=30 T=0.1，最终未被采用 |
| **RL 策略权重** | ❌ 缺失 | **高** | `general_ppo` 库 + `boost1.pth` 权重未包含，实际RL策略无法复现 |
| **Vision 模型权重** | ❌ 需邮件申请 | **高** | YOLO11s-seg 训练权重 + TensorRT engine，邮件 hc_yang@langyi.tech |
| **Vision 数据集** | ❌ 需邮件申请 | 中 | 语义分割训练数据集 |
| **机器人接口** | ⚠️ 仅框架 | 中 | ZMQ 通信骨架，IP脱敏，缺少完整双工协议 |
| **VO 角速度限制表** | ❌ 缺失 | 中 | `vo_angular_limits.csv` 被引用但不在仓库中 |
| **path_process.hpp** | ❌ 空文件 | 低 | 仅3行#ifndef，逻辑全在.cpp |
| **全局规划器** | ❌ 不存在 | — | 项目本身没有全局规划模块（非未开源，而是没有） |

### 关键影响

1. **视觉分割权重缺失** — 没有训练好的 YOLO11s-seg 模型，视觉感知链路完全断开
2. **RL 策略权重缺失** — RL增强控制无法复现，只能使用纯模糊控制
3. **MPC 被 disable** — 说明团队最终选择了模糊+RL方案，MPC仅供参考
4. **无全局规划** — 这是项目最大空白，需要自行补充

### 视觉与避障解耦的原因

代码中 Vision（DeepStream+TensorRT 语义分割）与 VO 避障（tangent_arc_navigation）当前是解耦的——视觉结果没有流入避障决策。原因：

**半马比赛场景特殊性：** 机器人半程马拉松有专用赛道，且逐个出发。机器人只需检测道路边界和静态障碍物，不需要实时避让行人或其他机器人。LiDAR 的 LaserScan 聚类已足够完成此任务。

**对四足巡线导航的启示：** 四足在开放环境中导航（公园、校园、街道），必须面对行人、车辆、其他机器人等动态障碍。**必须将视觉检测结果桥接到避障系统**，`ObstacleArray.msg` 格式天然兼容（bbox→polygon, mask→多边形），只需补一个 ROS topic 桥接节点。

## 六、四足机器狗适配路线图

```
Phase 1: 定位层移植 (1-2周)
├── glio_mapping 配置调整（禁用ZUPT、调IMU参数）
├── 选择 LiDAR + IMU 组合
└── 验证 ESEKF + 去畸变在四足晃动下的效果

Phase 2: 极简导航移植 (2-3周)
├── tangent_arc_navigation（VO避障）
├── 修改运动学包络（差分→全向）
├── 重标定速度范围和碰撞体
└── 纯模糊控制先跑通

Phase 3: 复杂导航移植 (3-4周)
├── local_planner（体素地图+路径采样+Dijkstra）
├── 训练四足专用的 RL 策略
└── 运动学包络适配

Phase 4: 室内外一体化 (3-4周)
├── 补充全局规划器（OSM/高德）
├── GNSS 退化检测 + 纯SLAM切换
├── 地图衔接（室内点云图 ↔ 室外GNSS图）
└── ROS2 迁移（可选但推荐）
```
