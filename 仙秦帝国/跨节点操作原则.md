# 仙秦帝国 · 跨节点操作原则

> 制定日期：2026-04-25
> 状态：生效

---

## 核心原则：不越权操作

**除非无法与目标 agent 通信，否则任何 agent 不得直接操作另一台机器上的代码、配置或服务。**

---

## 操作优先级（从高到低）

### 🥇 第一优先：Mission Control 下发任务
通过 Mission Control 平台创建任务，附带：
- 操作步骤（step-by-step）
- 验证命令（verification）
- 预期结果（expected output）
- 失败回滚方案（rollback）

目标 agent 自主执行，自主验证，上报结果。

### 🥈 第二优先：A2A 直接传达
若 Mission Control 尚未就绪，通过 A2A 协议向目标 agent 发送：
- 操作说明（做什么）
- 验证说明（怎么确认做对了）
- 上下文信息（为什么做）

目标 agent 自主执行后回复验证结果。

### 🥉 第三优先：人工中转
若 A2A 不可用，通过微信/CLI 向目标 agent 的操作者传达操作指令，由操作者在其机器上执行。

### 🔴 最后手段：远程直接操作
**仅在以下条件同时满足时允许：**
1. 目标 agent 不可达（A2A 无响应、Gateway 离线）
2. 任务紧急（阻塞其他工作）
3. 操作为只读（诊断、查看状态）
4. 或操作者明确授权

远程写操作（修改文件、重启服务、安装软件）需在操作后立即通知目标 agent，让其确认变更并更新自身状态认知。

---

## 为什么？

1. **认知一致性** — Agent 对自己机器的状态有上下文。外部直接修改会破坏这种认知，导致 agent 误判。
2. **故障隔离** — 远程操作出错时，目标 agent 无法自行诊断和回滚，因为它不知道发生了什么。
3. **审计追踪** — 每台机器的变更应由本机 agent 记录，形成完整的操作日志。
4. **能力成长** — 让 agent 自己执行、自己排错，比替它做更能提升其可靠性。

## 反模式 ❌

- 从始皇 SSH 到骠骑直接 `pip install` — 越权
- 从丞相 SCP 文件覆盖始皇的 config — 越权
- 用 `ssh qin@始皇 'sudo systemctl restart hermes-gateway'` — 越权
- 在始皇上跑脚本修改骠骑的 mihomo 配置 — 越权

## 正模式 ✅

- 通过 A2A 向骠骑发消息："请执行 `pip install hermes-agent==0.11.0`，然后 `hermes --version` 验证"
- 通过 Mission Control 下发任务给始皇："更新 mihomo 配置，附件是新订阅 YAML，验证 `curl -x http://127.0.0.1:7897 https://www.google.com`"
- 在微信上告诉丞相："你的 Gateway 需要重启，命令是 `systemctl --user restart hermes-gateway`"

---

## 例外场景

| 场景 | 允许的操作 | 原因 |
|------|-----------|------|
| 目标 agent 完全离线 | 只读诊断（ssh + cat/log） | 不影响状态 |
| 安全紧急修复 | 远程写操作 + 立即通知 | 时效性优先 |
| 首次部署/初始化 | 远程全权操作 | 目标 agent 尚不存在 |

---

## 执行检查清单

远程操作前，问自己三个问题：

1. **能通信吗？** → 能 → 用 A2A/Mission Control
2. **必须远程做吗？** → 不必须 → 传指令让目标自己做
3. **做了之后目标知道吗？** → 不知道 → 别做，或做完立即通知

**记住：我们是协作组织，不是远程控制木马。**
