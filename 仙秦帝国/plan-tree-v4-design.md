# Plan-Tree v4 — 决策点：自我的思考层

> 版本: 4.0 | 2026-04-27 | 萱萱 设计
> v3 → v4 核心改动：加一个字段 `决策:`，记录 OODA 中被省略的 O（Orient）
> 理论支撑：STReasoner (时空因果推理) + 君上"自我痕迹"哲学

---

## v3 缺失了什么

v3 记录了完整的动作轨迹：

```
流入 → 产物 → 判定 → 流出 → 关联 → 预测
```

这是**OODA 的 D-A 环**（Decide-Act）。但缺了 O（Orient）——赋予观察以意义，判断"这意味着什么，我应该做什么选择"。

| 人类 | v3记录了吗 |
|------|:---:|
| "编译失败" | ✅ 流入 |
| "这意味着依赖缺失" | ❌ |
| "有三个修复方案" | ❌ |
| "我选了改Dockerfile，没选手动安装" | ❌ |
| "因为Dockerfile可以复现" | ❌ |
| "编译通过了" | ✅ 判定 |

**缺失的是整个思考层。** 一个月后看 Plan-Tree，看到 `[✓] 编译通过`，看不到当时做了多少个方案、为什么选这个、是对是错。

## v4 新增：决策点

### 格式

```
- HH:MM 决策: 面对什么问题？选什么？
  分叉:
    A) 方案A — (优点/成本/风险)
    B) 方案B — (优点/成本/风险)
    ...（可以多于2个分叉）
  选择: A
  理由: 因为...（引用关联边/历史轨迹/预测数据）
  放弃: B（为什么不可取），C（为什么不选）...
```

### 示例

白起 NAV_DOG E2E 的决策点：

```
- 04-27 14:25 决策: E2E集成策略？
  分叉:
    A) 五模块全串联 — 验证完整架构，但接口不匹配风险
    B) 三模块简化版 — 更快出结果，但不证明完整闭环
    C) 先修超声再串 — 确定性高，但多花2h
  选择: A（五模块全串联）
  理由: 五个模块独立自测都PASS（见历史轨迹），接口匹配概率>70%
        即使A失败，降级到B只需1h
  放弃: C（2h代价太大，预测乐观路径今晚17:00可完成）
  关联: 如果A成功 → #navdog-deploy 今晚可启动（硬依赖解除）
```

### 决策点与判定点的闭环

```
决策: 选A，因为...
  ↓
产物: A执行后的结果
  ↓
判定: PASS ✅ → 验证了决策A正确    （绿色闭环）
       FAIL ❌ → 决策A错误，触发B    （红色闭环 → 新决策点）
```

## 映射 STReasoner

| STReasoner | Plan-Tree v4 |
|-----------|-------------|
| T1 因果溯源 — 谁导致了异常？ | 决策点 `选择: A 理由: 因为B被阻塞` |
| T2 实体识别 — 节点角色？ | 决策点 `关联: 如果A成功→解除B的硬依赖` |
| T3 相关性推理 — 如何传播？ | 决策点 `分叉: A→影响C, B→影响D` |
| T4 时空预测 — 未来怎样？ | 决策点 `理由: 预测乐观路径17:00完成` |
| S-GRPO — 有结构时推理更好 | 决策点 `理由: ...` 必须引用关联边 |

**核心：决策点不是"拍脑袋选一个"，是在时空结构中推理"选A的影响链是什么"。**

---

## v3 → v4 改动清单

| | v3 | v4 |
|---|-----|-----|
| 节点字段 | 流入/产物/判定/流出/关联 | + **决策**（放在流入之后、产物之前） |
| 节点可以有多少决策点 | 无 | **多个**（复杂任务可以有多个选择点） |
| OODA 映射 | D-A | **完整 O-O-D-A**（观察→判断→决策→行动） |
| 自我痕迹 | 只有行动 | **行动 + 思考** |
| 非强制 | ✅ | ✅（不留决策点就退化到 v3） |

---

## 为什么只加一个字段，不引入新机制

v3 的一切都保留：NOW 线、五种关联、预测区、预测归档。不需要新工具。

`决策:` 只是一个文本块。Agent 在做选择时顺手写下来——就像人类在做困难选择时会在心里过一遍"我为什么选A不选B"。不强制——快速决策（比如工具调用）不需要记录决策点。

**一个字段的增量，换来完整的自我痕迹。**
