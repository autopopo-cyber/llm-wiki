## 概述

多机编队（Multi-Robot Formation）让多台机器狗以特定几何队形协同移动。应用场景包括：
- **队列巡逻**：多台机器狗前后跟随，覆盖更大区域
- **物资搬运**：多台协同搬运大件物品
- **搜索与救援**：多区域并行搜索
- **护卫编队**：围绕人类形成保护队形

与集中式调度不同，本文档强调**去中心化编队**：机器人之间直接通信，不强制依赖云端或后台规划。

## 一、编队架构对比

### 1.1 集中式 vs 分布式 vs 去中心化

| 架构 | 通信方式 | 依赖网络 | 鲁棒性 | 适用场景 |
|------|---------|---------|--------|---------|
| **集中式** | 机器人↔云端 | 必需 | 低（单点故障） | 调度监控 |
| **分布式** | 机器人↔机器人+云端 | 部分可离线 | 中 | 任务协同 |
| **去中心化** | 机器人↔机器人（P2P） | **不需要** | **高** | **编队行走** |

本文档聚焦**去中心化编队**，作为机载能力的一部分。

### 1.2 两种编队模式

| 模式 | 原理 | 通信需求 | 复杂度 |
|------|------|---------|--------|
| **领航-跟随（Leader-Follower）** | 一台领航，其余跟随其相对位姿 | 低（仅接收领航状态） | 低 |
| **虚拟结构（Virtual Structure）** | 全队跟踪一个虚拟刚体的坐标 | 中（需共享虚拟坐标） | 中 |
| **行为基础（Behavior-Based）** | 每个机器人根据邻居行为自主调整 | 高（需感知邻居状态） | 高 |

**推荐组合：**
- **常规队列**：领航-跟随模式（简单可靠）
- **复杂队形**：虚拟结构模式（三角/菱形/圆形）

## 二、通信方案（机载 P2P）

### 2.1 无线通信选择

| 方案 | 协议/频段 | 距离 | 带宽 | 特点 | 推荐度 |
|------|----------|------|------|------|--------|
| **WiFi Direct** | 802.11 | 100m | 高 | 无需路由器，直接P2P | ⭐ 首选 |
| **ESP-NOW** | 2.4GHz | 200m | 低 | 超低延迟（<10ms），轻量 | ⭐ 辅助 |
| **UWB（超宽带）** | 3.5-6.5GHz | 50m | 中 | **厘米级测距+测角** | ⭐⭐ 最佳 |
| **蓝牙Mesh** | 2.4GHz | 30m | 低 | 自组网，适合近距离 | ⚠️ 低带宽 |

**最佳组合：WiFi Direct + UWB**

```
WiFi Direct：状态广播（位置、速度、意图）
UWB：精确测距测角（邻居相对位置到±10cm）
```

### 2.2 UWB 精准定位

**芯片选型：**
- **Decawave DW1000** / **Qorvo DWM1000**：开源驱动成熟
- **Apple U1 / Qorvo DWM3000**：新一代，精度更高

**定位原理：**
```
每台机器人携带 UWB 标签
邻居机器人作为锚点
TWR（双向测距）→ 获取距离
多锚点三角定位 → 获取相对坐标
⇒ 每台机器人知道：
   - 我相对于领航者的位置（±10cm）
   - 我相对于左右队友的位置
```

**无需 GPS/RTK，无需网络，纯本地 P2P。**

## 三、领航-跟随模式详解

### 3.1 编队几何定义

```
领航机（Leader）
    │
    │ 距离 d₁ = 2.0m
    │ 角度 α₁ = 180°（正后方）
    ▼
跟随机1（Follower 1）
    │
    │ 距离 d₂ = 2.0m
    │ 角度 α₂ = 180°
    ▼
跟随机2（Follower 2）
```

**期望队形 = {(d₁, α₁), (d₂, α₂), ...}**

### 3.2 跟随控制器

与单机器人跟人跟随的控制律相同，但目标变为领航机的历史轨迹点：

```
领航机在 t 时刻位于 P_L(t)
跟随机期望：P_F_target = P_L(t - τ) + R(α)·[d, 0]

τ = d / v预估（延迟补偿）
α = 期望角度（180°=正后方，150°=侧后方）
```

**控制输出：**
```
v_F = k_v · |P_F_target - P_F| + k_vd · v_L
ω_F = k_α · atan2(y_err, x_err) + k_ω · ω_L
```

### 3.3 队形变换

| 变换 | 触发条件 | 动画示意 |
|------|---------|---------|
| 一字纵队 → 横向展开 | 进入开阔区域 | `| → —` |
| 横向展开 → 三角防御 | 进入未知/危险区域 | `— → △` |
| 收缩圆环 | 围绕人类保护 | `— → ○` |
| 分批散开 | 搜索任务 | `— → · · ·` |

**变换过程平滑插值：** 期望 (d, α) 渐变过渡，时间常数 2-5 秒。

## 四、避障与编队协调

### 4.1 个体避障 vs 编队避障

| 层级 | 决策 | 范围 |
|------|------|------|
| **个体避障** | 单机器人绕开前方障碍 | 自身安全半径 0.5m |
| **编队避障** | 全队协同选择绕行通道 | 编队占据空间 |

**关键问题：** 跟随机绕障时可能阻碍领航机，或脱离队形。

### 4.2 编队避障策略

**策略 A：队形收缩通过窄道**
```
检测到窄道（宽度 < 编队展开宽度）
    ↓
领航机广播：收缩队形
    ↓
所有人计算新队形（一字纵队或更小间距）
    ↓
通过窄道后恢复
```

**策略 B：分流通过多障碍区**
```
障碍物密集到无法保持队形
    ↓
领航机选择最优路径
    ↓
跟随机各自独立避障，但保持向领航机收敛
    ↓
通过密集区后重新编队
```

**策略 C：领航机开路**
```
领航机负责探测前方可行域
    ↓
跟随机沿领航机的"已验证路径"行走
    ↓
类似于人类在雪地/丛林中跟随前人的脚印
```

### 4.3 与 vo_navigation 的集成

```
┌─────────────────────────────────────────────┐
│           单机器人避障层                       │
│      （vo_navigation：切弧避障）              │
│                                              │
│  输入：waypoints / 跟随目标 / 编队期望位置      │
│  输出：候选速度方向集合                         │
│                                              │
│  then:                                       │
│    ├─ 自由导航：选最接近期望的路径              │
│    ├─ 跟随模式：选朝向跟随目标的方向            │
│    └─ 编队模式：选保持队形的方向               │
│        + 避让队友（距离 > 0.8m）               │
└─────────────────────────────────────────────┘
```

**队友避让：** 将其他机器人视为动态障碍物（预测其未来位置，膨胀安全半径）。

## 七、技术决策记录

| 日期 | 决策 | 理由 |
|------|------|------|
| 2026-04-20 | 编队通信优先 P2P | 去中心化、高鲁棒、无网络依赖 |
| 2026-04-20 | UWB 做测距主传感器 | 厘米级精度、低延迟、不受光照影响 |
| 2026-04-20 | WiFi Direct 做状态广播 | 带宽充足、无需路由器、Android 原生支持 |
| 2026-04-20 | 领航-跟随为主模式 | 简单可靠、故障隔离（跟随机故障不影响全队） |
| 2026-04-20 | 队形变换渐进插值 | 避免急转导致跟踪丢失 |
